## 🖼️ Screenshot Creation Tips

### Tools Needed
- **Device Simulator**: Xcode Simulator or real device
- **Screenshot Tool**: Built-in iOS screenshot or third-party tool
- **Image Editor**: Figma, Sketch, or Photoshop for cleanup

### Best Practices
1. **Use Real Content**: Show actual reports, not placeholder text
2. **Clean Status Bar**: Full battery, good signal, clean time
3. **Consistent Lighting**: All screenshots in same mode (light/dark)
4. **No Personal Data**: Use fake but realistic data
5. **High Quality**: Crisp, clear images without compression artifacts

## 🎨 Design Guidelines

### Color Scheme
Use your app's existing colors:
- Primary Blue: #003A70
- Background: Clean whites/grays
- Status Colors: Green (fixed), Orange (in progress), Red (pending)

### Typography
- Use system fonts (SF Pro on iOS)
- Clear, readable text sizes
- Good contrast ratios

### Layout
- Follow iOS Human Interface Guidelines
- Consistent spacing and alignment
- Clear visual hierarchy

## 📱 Testing Screenshots

### Validation Checklist
- [ ] Screenshots load properly in App Store Connect
- [ ] All text is readable
- [ ] Images are crisp and clear
- [ ] No cut-off content
- [ ] Consistent with actual app appearance
- [ ] Professional and polished look

### Common Issues to Avoid
- Blurry or pixelated images
- Inconsistent device frames
- Personal/sensitive information visible
- Placeholder or "Lorem ipsum" text
- Screenshots that don't match actual app
- Poor lighting or contrast

## 🚀 Quick Start Commands

### Generate Screenshots with Flutter
```bash
# Install screenshot tool
flutter pub global activate screenshots

# Configure screenshots.yaml
# Run screenshot generation
screenshots
```

### Manual Screenshot Process
1. Open iOS Simulator
2. Run app in Release mode
3. Navigate to each key screen
4. Take screenshots (Cmd+S in Simulator)
5. Export at required resolutions
